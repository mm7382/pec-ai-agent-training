Report Brief

這堂課要解決的問題

  • 多數人同時使用 Claude、ChatGPT、Gemini、本地模型與多個 agent，但每個工具都像孤島，彼此沒有資料流、狀態與成果回饋。
  • 真正需要管理的不是單一 prompt，而是 token 花費、訂閱方案、知識庫新鮮度、skills 使用率、agent 工作紀錄與自動化改進建議。
  • 本課的核心是建立一個 Claude Agentic OS：把模型、知識庫、技能、agent 與自動化流程集中到可視化 dashboard。

教學對象

  • AI Agent / Hermes / Codex / Claude Code 使用者：已經有多個 agent 或工作流，需要集中管理的人。
  • 系統工程師與 PM：需要追蹤工具成本、工作流價值、專案活動、知識庫與自動化成果的人。
  • 內部工具開發者：想把 agent 能力包成可交付 dashboard，提供給團隊或客戶使用的人。
  • 不適合對象：只想看單一聊天工具示範，尚未累積任何 agent、skills 或知識庫的人。
Visual Roadmap

Claude Agentic OS 架構

01Layer 1

AI Usage Dashboard - 顯示 Claude、ChatGPT、Codex 等訂閱、token 使用、套餐額度與每月成本。

02Layer 2

Knowledge Hub - 串接 Obsidian、Pinecone、Notion 或本地記憶，讓知識庫狀態與關聯可視化。

03Layer 3

Skills Registry - 把所有 skills 集中管理，追蹤使用次數、有效性、冷門技能與可封存項目。

04Layer 4

Agent Activity Map - 彙整 Hermes Agent、Claude Code、Codex 等 agent 的任務紀錄與工作區活動。

05Layer 5

Dreaming Engine - 在固定時間自動掃描工作流，提出成本、技能、知識庫與專案結構的改進建議。

System Workflow

工程師操作模型

STEP 1Step 1

下載 OS template，放到 VS Code、Cursor 或任一 IDE 專案資料夾。

STEP 2Step 2

讓 Claude 依 `CLAUDE.md` 或專案指令安裝 dashboard 與本機服務。

STEP 3Step 3

設定個人/團隊資料、照片、時薪或每小時價值，讓系統能估算 AI 節省成本。

STEP 4Step 4

掃描本機工具與 CLI，例如 Claude App、Claude Code、Codex、VS Code、Antigravity 等。

STEP 5Step 5

連接知識庫與外部工具，例如 Obsidian、Pinecone、Notion integration token。

STEP 6Step 6

設定 Dreaming 排程，決定每天或每週自動掃描並產生改善建議。

Engineering Applications
1

成本雷達

找出高成本模型使用比例，例如把簡單任務從 Opus 類模型切到 Haiku 類模型以降低 token 花費。

2

技能盤點

找出 30 天未使用的 skills，建議移到 archive，降低系統負擔與選擇成本。

3

專案健康檢查

偵測高頻使用但缺少 `CLAUDE.md` 的工作區，提醒補上 onboarding 與常見任務規則。

4

知識庫視覺化

把 Obsidian、Pinecone、Notion 與 agent task log 連成一張可查看的關聯圖。

5

價值計算

根據使用時間、任務類型與時薪設定，估算 AI 工作流實際節省的金額。

Teaching Notes

常見誤解

  • 誤解 1：Agentic OS 是另一個聊天工具。實際上它是管理多個 AI 工具與工作流的控制台。
  • 誤解 2：只要有 Claude Code 就夠。影片重點是把 Claude Code、Codex、知識庫、skills 與成本資料放到同一個系統中。
  • 誤解 3：Dashboard 只是漂亮介面。真正價值在於看見成本、使用率、過期知識、閒置技能與可改善流程。
  • 誤解 4：Dreaming 是自動魔法。它仍需要清楚的資料來源、工作區規則與人工確認採納。

工作坊設計

  • 0-10 min：盤點目前團隊使用的 AI 工具、agent、知識庫與自動化腳本。
  • 10-25 min：畫出現有 AI 孤島，標記哪些工具沒有共享狀態、成本或任務紀錄。
  • 25-40 min：設計第一版 Agentic OS dashboard：成本、skills、知識庫、agent activity、Dreaming。
  • 40-55 min：用一個真實工作區建立 `CLAUDE.md` 或 agent instruction，讓系統能被自動掃描與改善。
  • 55-60 min：定義下週驗收：至少一個成本優化、一個技能封存、一個知識庫補強、一個 dashboard 指標。

驗收清單

  • 能說明 AI 工具孤島會造成什麼管理問題。
  • 能畫出 Agentic OS 的五層架構：Usage、Knowledge、Skills、Agents、Dreaming。
  • 能列出 dashboard 必須追蹤的指標：token、成本、訂閱、skills、專案活動、知識庫狀態。
  • 能設計一個 ADLINK 內部 agent 管理中心的最小可行版本。
  • 能判斷哪些改善建議可自動採納，哪些必須由 owner review。
