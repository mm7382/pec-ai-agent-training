Course Brief

一句話先記住：Chat 用來問，Cowork 用來交辦，Code 用來改專案。

Claude Desktop 現在把三種工作方式放在同一個桌面入口。新手最容易卡住的不是模型能力，而是不知道「這件事該放在哪個分頁做」。本教材會用你提供的三張畫面，逐一拆解每個按鈕與 SOP。

先選對模式：不要把所有事都丟進同一個聊天框

官方文件把三個分頁定義得很清楚：Chat 是一般對話，Cowork 是背景自主工作，Code 是能接觸專案檔案的互動式 coding assistant。

01
Chat

你想問、學、寫、整理想法

適合快速問答、翻譯、寫信、學概念、產生草稿、做 artifact。預設不直接讀你的本機專案檔。

Cowork

你想交辦一個可完成的任務

適合整理資料、研究、產出報告、排程任務、跨工具工作。它比較像背景工作者，你要給清楚目標與邊界。

Code

你想讓 Claude 看專案、改檔、跑命令

適合理解 codebase、修 bug、重構、寫測試、檢查 diff。它會接觸本機檔案，所以要懂權限與審查。

ADLINK 建議：新手第一週只做三件事：Chat 問概念、Cowork 做低風險報告、Code 只在測試 repo 或備份專案內操作。

Chat SOP：從空白對話開始

Chat 是最安全的起點。你可以先用它問概念、改文字、學工具，等任務變清楚再轉 Cowork 或 Code。

Chat 使用步驟
  1. 按 New chat，不要把新主題塞進舊對話。
  2. 先說角色與目的：例如「你是 ADLINK 內訓講師」。
  3. 貼上背景、限制與輸出格式。
  4. 要求 Claude 先問問題，不要立刻生成最終答案。
  5. 確認方向後，再請它輸出文件、表格、SOP 或 artifact。

A. 輸入框與右側控制
可點位置意思新手怎麼用注意事項
/Type / for skills 輸入主要問題。輸入 / 可叫出 Skills 或命令。 第一次使用先用自然語言，例如「請用 SOP 方式教我」。等熟悉後再用 / 找指定 skill。 不要把密碼、token、cookie、客戶個資直接貼進去。
+加號 加入附件、圖片、文件或其他上下文。 要請 Claude 看截圖、PDF、會議紀錄、規格書時使用。先說明「這份檔案要拿來做什麼」。 上傳前確認檔案沒有機密憑證；精確 diff 或正式法務/財務資料不要只靠摘要判斷。
Sonnet模型名稱 選擇本次對話使用的模型。截圖顯示 Sonnet 4.6。 一般寫作、整理、教學、分析可用 Sonnet。困難推理或大型規劃再考慮更高能力模型。 模型可用性會隨帳號、方案、官方狀態改變；畫面若顯示不可用，就改選可用模型。
Loweffort 推理/投入程度。Low 通常較快、消耗較低；Medium/High 較適合複雜任務。 改信件、摘要、翻譯用 Low；架構判斷、SOP、除錯分析用 Medium 或 High。 不要為了省用量讓複雜任務一直用 Low，品質可能不穩。
Mic麥克風 語音輸入，把你講的話轉成文字。 適合快速口述需求。送出前最好補上條列式驗收條件。 口述容易漏限制，正式交辦前要再看一次文字。
Wave聲波圖示 Voice mode / 語音對話入口。官方說手機端也會用聲波圖示進入 voice mode。 用來邊想邊聊、做口頭 brainstorming。 需要精準輸出時，最後仍要轉成文字 SOP 或表格。
Learn more提示連結 查看目前狀態或功能限制的說明。截圖中用來說明「Claude Fable 5 is currently unavailable」。 看到模型不可用、功能限制、beta 提醒時先點這裡看原因。 這不是主要工作按鈕，只是說明連結。
B. 下方快捷 chip：Write / Learn / Code / Life stuff / Claude's choice
快捷 chip代表任務適合場景建議 prompt 補法
WriteWrite 寫作、改寫、摘要、信件、報告草稿。 ADLINK 週報、教學講稿、客戶回覆草稿。 補上對象、語氣、字數、格式、不可提到的內容。
LearnLearn 學概念、做教學、把複雜內容拆成課程。 新工具導入、AI agent 觀念、內部訓練教材。 指定學員程度：「完全沒用過」「工程師」「PM」。
</>Code 問程式概念或產生程式片段。 先問「這段錯誤代表什麼」或「這個 repo 架構怎麼讀」。 若要讀本機專案、改檔、跑測試，應切到 Code 分頁。
CupLife stuff 生活、行程、個人整理類任務。 整理待辦、行程規劃、非公司機密的日常協助。 公司資料與私人資料不要混在同一段對話。
ChoiceClaude's choice 讓 Claude 自行判斷合適的回答方式。 你只知道目標、不確定要文章、表格或步驟時。 若任務有固定格式，還是要明講「請輸出表格/SOP/HTML」。

快捷 chip 只是起手式，不會替你補完上下文。真正影響品質的是你有沒有交代目的、資料來源、限制與驗收標準。

Projects / Artifacts：Chat 裡的兩個工作收納區

官方定義：Projects 是有自己聊天紀錄與知識庫的工作區；Artifacts 是可以在對話旁邊保存、修改、分享的獨立內容。

可點位置意思與用法

Customize / Directory：Skills、Connectors、Plugins 怎麼看

Customize 是把 Claude 變成「符合你工作方式」的地方。官方新 Directory 把 Skills、Connectors、Plugins 放在同一個入口，讓你搜尋、安裝、啟用或停用。

Cowork SOP：把任務交出去，但邊界要寫清楚

Cowork 是比較「自主」的模式。官方說它能處理多步驟任務，回來交付文件、整理結果或研究報告；但也因為自主與網路能力，安全邊界要更嚴格。

08
Cowork tab交辦長任務。適合報告、整理、研究、跨文件彙整，不適合一開始就交付高風險操作。
New task建立新的交辦任務。每個任務都要有明確目標、資料位置、輸出格式與停止條件。
Projects把 Cowork 任務放進專案，保留文件、指令與任務脈絡。
Artifacts查看或管理產出的文件、工具或視覺化成果。
Scheduled管理週期性任務。官方說可用於每日簡報、週報、定期研究；每次執行都是自己的 Cowork session。
Dispatch Beta畫面顯示的交辦/派工入口。使用前先確認任務是否低風險、可回復、可審查。
Work in a project指定任務在哪個 project/工作區內執行。建議 ADLINK 每個產品線或 agent 有獨立 project。
Ask任務模式下拉。若只是詢問，選問答；若要交付成果，改成明確任務描述。
Get to know Cowork新手 onboarding checklist。先完成通知、角色客製、排程任務理解，再做正式工作。
Task suggestionsOptimize my week、Organize screenshots 等範例。新手可模仿格式，但要改成 ADLINK 需求。
Cowork 使用步驟
  1. 按 New task。
  2. 選擇是否 Work in a project，讓任務有固定背景。
  3. 用一句話寫交付成果：例如「產出一份 ADLINK 內部週報」。
  4. 列出可讀資料夾、不可碰資料、輸出格式、截止時間。
  5. 要求它先列計畫，等你確認後再執行。
  6. 完成後檢查來源、摘要、附件、遺漏項與下一步。

- 不要讀取 Desktop、Downloads 或任何 .env 檔
- 不要寄信、不要刪檔、不要修改原始檔

輸出：
1. 本週異常
2. 風險與 owner
3. 下週待辦
4. 需要我確認的問題

請先列出你準備怎麼做，等我確認後再開始。
ADLINK 安全建議：Cowork 可以很能幹，但正式工作前先建立「專用資料夾」。不要讓它掃整台電腦，也不要把客戶合約、帳密、token、財務資料放進任務範圍。

Code 使用步驟

  2. 第一句先要求「讀取專案結構，不要改檔」。
  3. 讓 Claude 說明它看到的架構、測試方式、風險。
  4. 給一個小任務，例如修一個 bug 或補一個 test。
  5. 要求先列計畫，再改檔；改完跑測試。
  6. 看 diff、看測試輸出，再按 Accept edits。

以下案例來自 Anthropic 官方文件、Claude Code common workflows、Cowork use cases、Anthropic teams 實務報告與 GitHub 社群 workflow。每個案例都改寫成 ADLINK 內部可直接套用的版本。

網路實例 原始用途 ADLINK 可怎麼用 建議模式
Anthropic Data Infrastructure：用 dashboard 截圖排查 Kubernetes 問題 把監控畫面交給 Claude Code 協助判斷 pod IP address exhaustion，並產生命令。 遇到 lab server、CI runner、內部 agent 服務異常時，先貼監控截圖與錯誤 log，要求 Claude 只做診斷與下一步命令草稿，不直接執行。 Code / Chat
Anthropic Finance：plain text workflow 讓非工程師執行資料流程 財務同仁用文字描述「查 dashboard、跑 query、輸出 Excel」，Claude Code 代為執行並詢問必要日期。 PM 可寫一份 `weekly_issue_report.workflow.md`，描述輸入資料夾、統計欄位、輸出 Excel/HTML 格式，交給 Cowork 或 Code 產出草稿。 Cowork

Anthropic end-of-session documentation updates 每次工作結束後，讓 Claude Code 摘要本次成果並建議更新 Claude.md，形成改進循環。 每次 Codex 或 art agent 任務結束，要求 Claude 產出「可沉澱成 skill 的規則、踩坑、下一次 SOP」。 Code / Chat
Claude Code common workflows：了解陌生 codebase 官方提供快速 overview、架構模式、資料模型、auth flow 等探索 prompt。 接手新 repo 時先問「入口、資料流、測試命令、危險檔案、最該先讀的 5 個檔案」，不要一開始就要求修改。 Code
Claude Code common workflows：修 bug、重構、測試、PR、文件 官方把日常開發拆成短 recipe，包含 plan before editing、work with tests、create PR、documentation。 ADLINK 工程任務應拆成「先計畫、再改一小段、跑測試、產 PR summary」；不要用一句「幫我修好」。 Code
Dynamic workflows：多 subagent 做 audit / migration / cross-checked research 官方建議當一個對話無法協調大量代理時，用可重跑的 script 編排 subagents。 大型 ADLINK 知識庫清理、跨 repo 安全掃描、產品競品研究，可用多 agent 分工，再由 reviewer 合併與查錯。 Code / Cowork
Cowork file organization：整理 Downloads 或資料夾 官方例子包含依類型與日期整理大量檔案、批次命名。 把測試照片、截圖、log dump 放進專用資料夾，要求 Cowork 依日期、產品、issue 類型分類，但禁止刪檔。 Cowork
Cowork expense/report：收據與雜亂輸入轉正式文件 官方例子包含處理 receipts、voice memos、scattered notes，輸出 polished document。 把客訴筆記、會議紀錄、Teams 摘要整理成「異常、影響、owner、deadline、下一步」。 Cowork
Cowork research synthesis：網頁、文章、paper、notes 綜合 官方例子是把 web search、articles、papers、notes 合成報告。 做 AI 工具導入、Claude Desktop SOP、Headroom 評估時，要求 Cowork 先列來源，再產 Evidence Notes，不直接出結論。 Cowork / Chat
Cowork scheduled tasks：daily briefing / weekly report / recurring research 官方說 scheduled tasks 可產生日簡報、週報、定期研究、檔案整理、team updates。 建立每週 AI agent 維護報告：檢查模型、磁碟、未提交知識、失敗 tunnel、重複檔案，再寄草稿給你確認。 Cowork Scheduled
GitHub TDD workflow：Red → Green → Refactor 社群 guide 強調要明確要求 Claude 先寫 failing test，不然它容易先寫實作再補測試。 在 Hermes agent 改功能時，用「先寫會失敗的測試，不要寫實作」逼出驗收條件，再讓 Code 實作最小通過版本。 Code
真正有用的範例不是「Claude 很厲害」，而是「輸入資料在哪、允許做什麼、輸出什麼、怎麼驗收」。ADLINK 內部 SOP 都應該用這四格描述。

安全與權限：新手最該先學的不是技巧，是邊界

官方安全文件提醒：Cowork 因為 agentic 與 internet access 有特殊風險；Code 因為能改檔、跑命令，也必須審查。

14
  • 不要上傳 `.env`、token、cookie、SSH key、憑證、客戶合約、財務資料。
  • 需要讀本機檔案時，建立專用工作資料夾，不要讓它掃整台 Mac。

  • Scheduled task 只做低風險摘要與提醒，先不要做不可逆操作。
  • 看到模型不可用、權限提示或奇怪行為，先停下，不要硬按允許。
  • 正式交付前，用人類審查來源、邏輯、格式與是否符合 ADLINK 規範。
