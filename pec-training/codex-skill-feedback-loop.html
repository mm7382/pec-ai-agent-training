核心概念與名詞解釋

核心問題：本地 agent 學到的東西會流失
本地 agent 每次被使用者修正後，其實都得到一段 know-how；如果沒有回寫成 Codex Skill，下一次 Codex 還是要重新被教一次。
核心概念
  • 要收集的不是聊天內容本身，而是可重複的方法：觸發條件、操作順序、禁用事項、輸出格式與品質檢查。
  • 每次人工修正都應被視為訓練資料：哪裡重複、哪裡誤判、哪個偏好要永久保留。
  • 目標是把一次性的使用經驗，整理成 Codex 下次能自動使用的工作規則。
教學提示：art agent 被你修正過 ADLINK logo、禁用字、手機版面與寄信流程後，這些規則不能只留在對話裡，要進 Codex Skill。
為什麼回寫到 Codex Skill
OpenAI 官方資料把 Skills 定義成可重複、可分享的 workflow，且可包含 instructions、examples 與 code；這正適合承接 agent 累積的 SOP。
核心概念
  • Skills 支援 Codex 與 API，代表這些 workflow 不只存在 ChatGPT 介面，也可以進入工程 agent 工作流。
  • Skills 遵循 Agent Skills open standard，可從一個產品下載後安裝到另一個產品。
  • Codex 是寫 code、review code、ship code 的 agent；把 SOP 寫成 skill，等於讓 Codex 先讀懂你的工作方式再動手。
教學提示：把「產 ADLINK HTML 前先查官方資料、禁止顯示來源時間戳、手機版不能水平溢出」寫成 skill，比每次口頭提醒穩定。
回寫流程：從修正紀錄變成 Skill
不要直接把對話貼進 skill；要先萃取成可執行規則，再放進 Codex Skill 的 trigger、steps、checks 與 examples。
核心概念
  • Trigger：什麼任務要使用這個 skill，例如來源連結轉 ADLINK 教材、PR review、客戶問題摘要。
  • Steps：固定流程，例如先取得來源、再做 evidence notes、再上網查官方資料、最後渲染 HTML。
  • Checks：品質閘門，例如繁體中文、ADLINK logo、手機版無溢出、不得套錯模板。
教學提示：每次你說「這頁不用出現」、「不要用來源流程字眼」、「要上網找資料」，都應被歸納成 renderer rule，而不是只改單一檔案。
好處一：Codex 更懂使用者
當 Codex Skill 裡有使用者偏好、公司格式、禁用字與完成標準，Codex 每次開始任務時就有更準確的上下文。
核心概念
  • 使用者不用重複解釋「我要 ADLINK 格式、繁體中文、手機可讀、不要來源時間戳」。
  • Codex 可以更快判斷哪些內容該放進教材，哪些只是內部處理流程不能顯示。
  • 這比單純依賴一次 prompt 更穩，因為 skill 會在相同任務反覆被套用。
教學提示：未來你丟新連結時，Codex 應自動知道：來源連結只是框架，實際教材要查官方資料後重寫成可教內容。
好處二：Skill 會越用越準
每次本地 agent 成功或失敗，都可以回寫成 skill 測試案例；Codex Skill 不只是保存規則，也能持續優化。
核心概念
  • 成功案例變成 examples，讓 Codex 知道什麼叫合格輸出。
  • 失敗案例變成 anti-patterns，讓 Codex 避免舊模板、簡體字、長條式手機版與重複段落。
  • 品質檢查可以變成固定 checklist，讓每次產出都先自查再交給使用者。
教學提示：這次你指出內容重複，下一版 skill 就要加入規則：同一個觀念只能在大綱、細節或總結中各出現一次，不要整頁反覆貼。
好處三：團隊 know-how 可以共用
Skills 可以分享給 workspace，也可依 open standard 在產品間移動；這讓個人 agent 經驗有機會變成團隊標準流程。
核心概念
  • 一個人修正過的流程，不必只存在自己的聊天紀錄或本機 agent。
  • 團隊可以共用教材產出、PR review、issue triage、客戶摘要等高頻 skill。
  • 共享前要保留權限與資料邊界：不要把客戶機密、token、內部未公開資訊直接放進 skill。
教學提示：ADLINK 可以把 PM、FAE、SIE、教材產線各自的最佳做法收斂成不同 skill，讓新人與 agent 都照同一套標準做事。
ADLINK 落地做法
先從 art agent 教材產線開始，把你已確認的偏好回寫成 Codex Skill；穩定後再擴到 PR review、問題分類與客戶摘要。
核心概念
  • 第一版：ADLINK 教材 skill，固定處理來源理解、官方資料補充、手機版 HTML、Telegram 預覽與寄信確認。
  • 第二版：Codex review skill，固定讀 AGENTS.md、檢查風險、跑測試、輸出問題清單。
  • 第三版：Agent skill registry，追蹤每個本地 agent 哪些 know-how 已回寫、哪些還只是臨時 prompt。
教學提示：衡量方式不要只看產出速度，也要看重工次數、人工修正次數、禁用字違規、手機版可讀性與寄信成功率。

好處與優化閉環

不用每次重講規則

使用者偏好、公司格式、禁用字與完成標準進入 skill 後，Codex 每次啟動任務就先讀到同一套規則。

內容重點研究依據

把失敗變成 anti-pattern

套錯模板、內容重複、手機版溢出、顯示來源流程字眼，都要變成下一版 skill 的禁止條件。

內容重點研究依據

把成功變成 examples

被使用者確認可以的 HTML、寄信流程、ADLINK logo 規則與口語化寫法，整理成範例讓 Codex 重複使用。

落地：再擴到 PR review、問題分類與客戶摘要，讓 Codex 依 ADLINK 的檢查方式、輸出格式與風險語言執行。
