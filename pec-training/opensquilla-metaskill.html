課程大綱

OpenSquilla 是什麼

OpenSquilla 是一個以 token efficiency 為核心的 open-source AI agent runtime，主張在相同預算下提高 agent 的任務密度與穩定性。

OpenSquillaAI Agent RuntimeMicrokernel

Token efficiency 與 smart routing

OpenSquilla 的主軸不是單純換更強模型，而是讓 agent 判斷任務複雜度，將簡單任務交給低成本模型，複雜任務才使用高階模型與更深 reasoning。

核心概念與名詞解釋

OpenSquilla 是什麼
OpenSquilla 是一個以 token efficiency 為核心的 open-source AI agent runtime，主張在相同預算下提高 agent 的任務密度與穩定性。
核心概念
  • 定位是 microkernel AI agent：核心引擎負責 orchestration 與狀態管理，其他能力透過 provider、memory、tools、MCP、sandbox、scheduler、skills 等模組延伸。
  • 產品重點包含 smart routing、persistent memory、secure sandbox、built-in search、local embeddings，以及多 channel 介面。
  • 適合拿來講解 agent framework 如何從單次聊天，走向可配置、可治理、可重複執行的 runtime。
教學提示：上課可用「作業系統核心 + 外掛能力」比喻：核心保持小而穩，工具、模型、記憶、通道與技能在外層擴充。
Token efficiency 與 smart routing
OpenSquilla 的主軸不是單純換更強模型，而是讓 agent 判斷任務複雜度，將簡單任務交給低成本模型，複雜任務才使用高階模型與更深 reasoning。
核心概念
  • Smart Routing 會依任務特徵選模型 tier，避免所有問題都用最高成本模型處理。
  • Hybrid feature analysis 結合手工特徵與 embedding/semantic feature，評估任務長度、語言、程式碼、關鍵字與複雜度。
  • Adaptive prompts 與 on-demand skills 讓 agent 只載入必要能力，降低 context 浪費。
教學提示：可用工程流程比喻：一般詢問走 quick check，跨檔案分析或報告產出才升級成 full analysis。
Memory 與 local embeddings
OpenSquilla 強調 durable memory，不把所有歷史對話塞回 prompt，而是把可回憶的使用者、專案與經驗資料放進可搜尋的記憶系統。
核心概念
  • 四層記憶可理解為 working memory、episodic memory、semantic memory、raw audit base。
  • Hybrid search 同時支援語意向量與全文關鍵字搜尋，讓 agent 能從過去資料找回有用 context。
  • Local embeddings 代表部分語意檢索可在本機完成，資料邊界比全雲端更容易控管。
教學提示：對工程團隊來說，memory 可存專案規則、常見測試流程與偏好輸出格式，但敏感資料要有明確邊界。
MetaSkill 是什麼
MetaSkill 是 OpenSquilla 的 reusable workflow protocol，用來把多步驟工作包裝成可觸發、可審計、可改進的任務協定。
核心概念
  • Skill 適合單一任務模式；MetaSkill 則適合包含多個 skills、tools、LLM calls、檢查點與輸出步驟的高價值工作。
  • MetaSkill 不創造新的執行原子，而是定義如何組織既有 skills、tools、LLM calls 與 sub-agents。
  • 設計目標是讓高價值工作從每次重新推理，變成可重複、可檢視、可 replay 的 protocol。
教學提示：可以把 MetaSkill 比喻成 Makefile：它不取代工具，而是規定工具如何被組合、檢查與交付。
MetaSkill request 的四個要素
高品質 MetaSkill request 需要 outcome、context、standard、boundaries，否則 agent 容易產生泛泛說明或越界行為。
核心概念
  • Outcome 說明要收到什麼產物，例如 decision memo、research report、project plan。
  • Context 提供材料、對象、時間範圍與限制條件。
  • Standard 定義什麼叫做好結果；Boundaries 則列出不可捏造、不可執行、需確認的行為。
教學提示：上課可讓學員把「幫我研究一下」改寫成含 outcome、context、standard、boundaries 的正式 request。
安全與治理：reviewable output
MetaSkill 與 agent workflow 的輸出應被視為可審查工作產物，不是直接替人做最終決策或執行高風險操作。
核心概念
  • OpenSquilla 文件強調 MetaSkill output 是 decision-support material 或 work-product draft。
  • 發布、安裝、付款、簽署、寄送訊息、修改 production system 等動作仍需明確授權。
  • 實務導入時，要把 permission profile、approval、sandbox 與 replay 一起納入課程。
教學提示：可用 ADLINK 內部流程比喻：agent 可以整理 issue 與建議 action，但正式發信、改 BOM、提交報告仍需負責人確認。
