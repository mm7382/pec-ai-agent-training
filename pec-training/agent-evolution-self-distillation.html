如何讓自己的 Agent 越用越懂你

自我蒸餾、Skill 化與定期自動化

給工程師、PM 與團隊主管的導入教學：把日常 agent 使用經驗沉澱成可重複、可審核、可改善的工作能力。

Learning Goal

這堂課要解決什麼問題

For engineers / PM / team leads
3 個核心能力

自我蒸餾、流程 skill 化、定期自動化，是讓 agent 從「會回答」進化成「會累積經驗」的基礎。

很多 agent 一開始看起來很聰明，但每次都要重新解釋背景、重新提醒格式、重新修正錯誤。真正能長期使用的 agent，需要把使用者修正、成功流程、失敗教訓與固定任務沉澱下來。

核心觀念

Agent 不是越裝越強，而是越能把工作經驗變成可重用的流程、知識與檢查標準，才會越用越穩。

Main Framework

Agent 進化的三層架構

1. 自我蒸餾 把每天的互動、修正、偏好、失敗經驗整理成文件。這是 agent 的學習原料。
2. 重複流程變成 Skill 把固定工作方法寫成可重用的 SKILL.md 或流程規範，讓 agent 下次不用重新摸索。
3. 自動化定期執行 把固定整理、週報、知識更新、候選 skill review 做成排程，但保留人工核准。

Operating Flow

標準導入流程

01 收集訊號 保存任務紀錄、修正意見、常見失敗、重複需求與輸出格式。
02 每日蒸餾 把紀錄整理成偏好、流程、風險、候選 skill 四類。
03 產出草稿 先產生 skill 草稿或流程文件，不直接啟用。
04 人工審核 確認適用場景、限制、風險與是否真的能減少重複工作。
05 啟用能力 核准後才安裝 skill、更新 agent 設定或加入自動化。
06 定期回看 每週檢查使用率、重複性、錯誤率與是否需要停用。

Examples

哪些流程適合變成 Skill

報告格式 週報、issue summary、測試摘要、HTML 教材輸出格式固定時。
資料整理 每次都要讀 Excel、log、會議紀錄、wiki 頁，再整理成相同結構時。
品質檢查 需要固定檢查來源、日期、風險、隱私、繁體中文與格式一致性時。
工具選擇 常常需要判斷何時用瀏覽器、文件工具、CodeGraph、記憶或本地 LLM 時。

Implementation Plan

建議四週落地計畫

第 1 週

建立蒸餾日報

每天產生一份簡短紀錄：今天處理什麼、使用者修正什麼、哪些需求重複、哪些錯誤值得避免。

第 2 週

整理候選 Skill

從蒸餾日報中挑出高頻流程，寫成 skill 草稿。每份草稿都要包含觸發條件、步驟、輸出格式與禁用事項。

第 3 週

建立核准機制

不要讓 agent 自動改自己的能力。新增、刪除、修改 skill 前，必須有人確認用途與風險。

第 4 週

接入定期自動化

把日報、週報、skill review、知識庫更新做成排程。排程只負責產生提案，不直接做高風險變更。

Governance

安全治理：自動化不等於自動亂改

可以自動做

  • 整理每日使用紀錄。
  • 產生候選 skill 草稿。
  • 產生週報與知識更新提案。
  • 標記重複流程與常見錯誤。

需要人工核准

  • 安裝外部工具或 skill。
  • 刪除或停用既有流程。
  • 修改 agent 權限、token、排程或本機服務。
  • 把未審核內容寫入正式知識庫。
常見誤解

Agent 進化不是讓系統自己決定一切，而是把「可自動整理」與「必須人工核准」分清楚。這樣才能提升效率，又不犧牲安全與可追溯性。

Template

一份好 Skill 草稿應該包含什麼

使用時機 清楚描述何時該用、何時不該用，避免 agent 每件事都亂套同一個流程。
標準步驟 列出 3 到 7 個可執行步驟，包含讀取資料、分析、輸出與驗收。
品質檢查 定義輸出前必須檢查的項目，例如來源、日期、隱私、格式、語言與風險。

Quality Gate

導入驗收清單

  • 是否真的減少重複說明：導入後，同類任務是否不用再重講格式與步驟。
  • 是否可追溯：每個新 skill 都能追到來源任務、修正紀錄或失敗教訓。
  • 是否有人工核准：高風險動作不由 agent 自行決定。
  • 是否能回滾：每次能力變更都知道改了哪個檔案、影響哪些流程。
  • 是否保護敏感資料：蒸餾內容不得包含 token、API key、cookie、session 或個人憑證。
  • 是否持續評估：每週檢查哪些 skill 有用、哪些重複、哪些應停用。

Final Takeaway

帶回工作的做法

先做小型試點，再擴大成制度

先挑一個固定流程，例如週報整理、HTML 教材輸出或 issue summary。連續一週做自我蒸餾，第二週產生 skill 草稿，第三週人工核准啟用，第四週再接自動化。這樣最穩，也最容易驗收。
