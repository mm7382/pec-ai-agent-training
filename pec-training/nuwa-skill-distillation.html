核心概念與名詞解釋

Nuwa Skill 在做什麼
Nuwa Skill 的目標很直接：把一個人或一個領域的思考方法，整理成 agent 可以反覆使用的 skill。
核心概念
  • 它基於 Agent Skills 協議，不只是一段 prompt，而是一個可以安裝的 skill 資料夾。
  • GitHub README 提到，它可用在 Claude Code、Codex、Cursor、OpenClaw、Hermes 等多種 runtime。
  • 對團隊來說，重點不是模仿名人，而是把專家方法變成可重複使用的工作能力。
教學提示：可以把它想成：把一位資深同事的判斷方式，整理成新人也能叫得出來的 AI 工作助手。
蒸餾不是叫 AI 演戲
角色扮演通常只是換口氣說話；蒸餾要整理的是這個人怎麼想、怎麼判斷、什麼事情不會做。
核心概念
  • 普通 role play 容易變成套話，看起來像，但沒有真正的判斷邏輯。
  • 蒸餾後的 skill 要能說清楚：遇到問題時先看什麼、怎麼取捨、什麼情況要保留不確定。
  • 這也是為什麼 README 把誠實邊界列為必要內容。
教學提示：判斷好壞的方法很簡單：如果只是換個語氣，那是扮演；如果判斷路徑也變了，才接近蒸餾。
先收資料，再做判斷
Nuwa Skill 的流程會先找資料，再抽規則。它不是直接叫模型憑印象回答。
核心概念
  • README 描述會看著作、訪談、社群內容、批評者視角、決策紀錄與時間線。
  • 這樣做的目的，是避免只抓金句或單一案例。
  • 真正有用的是穩定出現的判斷方式，不是漂亮句子。
教學提示：如果要蒸餾 ADLINK 資深工程師，就應該看設計 review、debug 紀錄、客戶問題與失敗案例，而不是只看簡報。
一個好 skill 要有五層
Nuwa Skill README 把蒸餾拆成五層：怎麼說話、怎麼想、怎麼判斷、什麼不做、知道局限。
核心概念
  • 說話方式只是表層，不能只做語氣模仿。
  • 心智模型與決策規則才是核心，因為它們決定答案怎麼來。
  • 反模式與局限很重要，因為它們告訴 agent 什麼時候不要亂答。
教學提示：工程 debug skill 不能只會回答，還要知道不能亂猜 root cause，不能跳過 reproduction。
做完還要測
Nuwa Skill 的方法不是產出就結束。README 提到會用公開回答過的問題測方向，再用沒討論過的問題測邊界。
核心概念
  • 方向測試：看答案是不是抓到核心思路。
  • 邊界測試：看資料不足時，會不會裝懂。
  • 觸發規則：避免 skill 在不適合的情境被亂用。
教學提示：在 ADLINK 內部導入時，每個 skill 都應該有測試題、失敗樣本與版本紀錄。
對 ADLINK 可以怎麼用
這套方法最有價值的地方，是把專家經驗變成新人能用、團隊能維護的工作流程。
核心概念
  • 可以整理資深 PM 的需求澄清方式、FAE 的客戶診斷方式、SIE 的 issue triage 方法。
  • 內部資料要先處理授權、隱私與保密邊界。
  • 交付物不應只是聊天人格，而是 checklist、判斷流程與可重複 prompt。
教學提示：可以先挑一個高頻場景做，例如客訴初判、規格澄清、測試報告 review。
