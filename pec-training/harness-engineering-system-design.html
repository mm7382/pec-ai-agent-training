核心概念與名詞解釋

Harness 是什麼
Harness 不是更長的 prompt，而是包在模型外面的工作系統：它提供工具、記憶、限制、回饋與檢查，讓 agent 能穩定完成任務。
核心概念
  • Prompt 解決「怎麼問」；Context 解決「給什麼資料」；Harness 解決「agent 在什麼環境裡做事」。
  • OpenAI 的案例把重點放在環境、規則、工具與回饋迴圈，而不是只靠模型自己變聰明。
  • 對工程團隊來說，Harness 就是把工作流程系統化，讓 agent 有路徑、有工具、有界線。
教學提示：可以把 Harness 想成工程師旁邊的一整套工作台：文件、工具、測試、log、review 規則都準備好，agent 才能做穩。
為什麼不是只做 Prompt Engineering
Prompt 只能告訴模型要做什麼；Harness 會把任務拆解、工具使用、狀態觀測與錯誤修正放進流程。
核心概念
  • OpenAI 提到，人類工程師的角色變成設計環境、指定意圖、建立 feedback loops，讓 Codex agents 能可靠工作。
  • Anthropic 的 harness design 也強調 generator 與 evaluator 分工，讓一個 agent 做事，另一個 agent 更嚴格地評分與回饋。
  • 這代表 AI 工程的重點從「寫好一句話」升級成「設計一個會自我修正的工作系統」。
教學提示：同一句需求，如果沒有工具與檢查，agent 可能只產出漂亮草稿；有 Harness，agent 會跑測試、看錯誤、改到通過。
Harness 的核心組件
一個實用 Harness 通常包含 Planner、Tools、Memory、Evaluator、Quality Gate 與 Iteration Loop。
核心概念
  • Planner：先拆任務與定義完成條件，避免 agent 一開始就亂做。
  • Tools：讓 agent 能讀檔、跑測試、查 log、操作瀏覽器或呼叫 API。
  • Evaluator：用標準檢查結果，並把具體問題回饋給 generator。
教學提示：做 ADLINK 教材時，Planner 定義章節，Tools 抓來源與官方資料，Evaluator 檢查繁體、手機版、ADLINK logo 與內容是否重複。
OpenAI 的重點：讓 Codex 看得懂系統
OpenAI 的 Harness Engineering 文章強調 agent legibility：把文件、工具、log、metrics、測試與架構規則放到 Codex 可讀的位置。
核心概念
  • OpenAI 提到 repository knowledge 要成為 system of record，AGENTS.md 更像目錄，不是一本塞滿規則的大百科。
  • 把 log、metrics、browser state、測試結果暴露給 Codex，能讓 Codex 自己觀察、修正、再驗證。
  • 人類的 taste、review comment 與 bug feedback 應持續回寫成文件、工具或檢查規則。
教學提示：對 ADLINK 來說，不是每次口頭提醒 agent，而是把教材規則、產品資料、版型要求與檢查腳本放在 Codex 找得到的位置。
Anthropic 的重點：Generator 與 Evaluator 分工
Anthropic 的案例說明，讓同一個 agent 自己判斷自己常會太寬鬆；把生成與評分分開，回饋會更有效。
核心概念
  • Evaluator 要有明確評分標準，例如設計品質、功能、可用性、code quality。
  • Evaluator 可以實際操作頁面、看結果、寫出具體 critique，再讓 generator 迭代。
  • 這個模式適合用在 HTML 教材、前端 UI、測試修正、客戶文件等需要品質判斷的任務。
教學提示：art agent 產 HTML 後，不應只看是否有檔案；還要有 evaluator 檢查手機可讀、內容不重複、沒有禁用字。
AutoHarness 的啟發：小模型也能靠系統變強
AutoHarness 論文指出，小模型可透過迭代回饋自動合成 code harness，避免非法動作，甚至在某些任務超過更大的模型。
核心概念
  • 論文摘要提到，Gemini-2.5-Flash 的許多失敗來自非法動作；harness 能把外部規則變成可執行防護。
  • 重點不是模型本身最大，而是能否把環境規則、限制與修正流程寫成 code 或 policy。
  • 這對本地 agent 很重要：本地模型不一定最強，但如果 Harness 設計好，仍可在特定任務上穩定輸出。
教學提示：本地 art agent 不必靠最強模型硬猜，只要有來源取得、官方資料查詢、ADLINK renderer、品質閘門與 Telegram 回饋，就能穩定產教材。
ADLINK 可以怎麼導入
先從高頻流程做 Harness，不要一開始做大平台；教材產線、PR review、issue triage 都適合先做。
核心概念
  • 教材 Harness：來源理解、官方資料補充、ADLINK HTML、手機預覽、寄信確認。
  • 工程 Harness：讀 AGENTS.md、跑測試、查 log、輸出風險與修正建議。
  • 客戶問題 Harness：分類 issue、補上下文、比對既有案例、產出可追蹤 action items。
教學提示：第一版先把 art agent 做成完整 Harness：產出後由你確認，確認過的修正再回寫成下一版規則。

系統閉環重點

環境比提示更重要

Prompt 只能描述需求；Harness 會把工具、規則、文件、log、測試與回饋變成 agent 能持續使用的環境。

內容重點研究依據

Generator 與 Evaluator 分工

一個 agent 負責產出，另一個 agent 依標準檢查結果並給具體 critique，比自我評分更容易收斂品質。

內容重點研究依據

把規則寫成可執行防護

AutoHarness 的啟發是：限制與規則不要只寫在說明裡，能變成 code、policy 或 quality gate 就更穩。
